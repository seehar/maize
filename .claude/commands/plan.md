---

为以下任务设计实现方案: $ARGUMENTS

## 规划步骤

1. 分析任务需求，明确目标和约束
2. 阅读相关代码，理解当前架构（确认影响 Classic/Lite/共享模块）
3. 设计实现步骤，列出需修改的文件清单
4. 评估每步风险等级（低/中/高）
5. 制定测试策略
6. 如有替代方案，简述取舍

## 输出格式

按以下结构输出：

```
# 实现计划: [功能名称]

## 背景与目标
## 影响范围
## 实现步骤（含文件路径和风险评估）
## 测试策略
## 替代方案（如有）
```
