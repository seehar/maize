# AGENTS.md

本项目使用 CLAUDE.md + `.claude/rules/*.md` 定义开发规范，本文件补充团队协作流程。

## 开发流程

### 任务组织

1. **若改动涉及 ≥3 个文件，必须先拆分任务**
   - 每个任务独立可测试、独立可部署
   - 避免"一锅端"的大 PR

2. **每个任务必须符合单一职责原则**
   - 一个 PR/提交只做一件事

3. **禁止无关改动和"顺手优化"**
   - 不要在修改 A 功能时顺便优化 B 功能
   - 不要在修复 bug 时顺便改格式

### 设计准则

1. **设计清晰度与可测试性优先于性能**
   - 先让代码正确可读，再优化性能瓶颈

2. **禁止过度抽象，仅在必要时引入**
   - 不要为了"可扩展"而过度设计

3. **需求不清晰时必须先提问**
   - 不猜测需求，不"我觉得应该这样"

## 爬虫模式开发边界

### Classic 模式
- 适用于需要中间件、管道、调度器的复杂场景
- 修改 Classic 模块时必须确保 middleware/pipeline 链路完整
- CrawlerProcess 负责生命周期管理，Spider 不应直接控制引擎

### Lite 模式
- 适用于简单抓取，不依赖中间件和管道
- Lite 模块应保持轻量，禁止引入 Classic 模式的复杂依赖
- 新增 Lite 功能时评估是否应放在 Classic 模式实现

### 共享模块
- `maize/base/` 和 `maize/common/` 为两种模式共享
- 修改共享模块时必须同时验证 Classic 和 Lite 模式不受影响

## 异步代码注意事项

- 所有 IO 操作必须使用 `async/await`，禁止在 async 函数中调用阻塞 API（如 `requests`、`time.sleep`）
- `aiohttp.ClientSession` 必须复用，禁止在循环中反复创建
- 异步任务必须正确 `await`，避免 fire-and-forget 导致异常被吞
- 连接和 session 必须在使用完毕后正确关闭（推荐 `async with`）

## Git 协作

- 主分支：`main`
- 提交前确认 `git status --short` 干净且测试已通过
- 优先按单一主题拆分提交

## 安全与配置

详细安全规则见 `.claude/rules/security.md`，核心原则：
- 不要把密钥、固定环境地址或生产专用参数硬编码到代码中
- 优先通过环境变量注入
